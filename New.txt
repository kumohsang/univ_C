1) 용어 설명

   int       test       = 123;
   타입          변수            초기화

2) 포인터
   int    test    = 123;   
   인티저
   
   int*    pTest    = &test;   // test 의 주소를 담는 포인터 변수
   인티저 포인터
   
   포인터 : 주소를 담는 변수이다.
   
   &   : 주소를 얻는 것
   *   : (역참조) 값을 얻는 것
   
   int*    pTest    = &test;
   test 라는 값을 담고 있는 int 타입 변수의 주소를    (주소를 얻을때 &를 활용!!)
   int* 타입의 pTest 라는 변수에 담는다.
   
   int    test    = 123;
   int*    pTest    = &test;
   
   test 는    123
   *pTest 도    123                   (값을 얻을때 역참조 !!)
   

3) 구조체 멤버
   struct stTest_t
   {
      int a;   // 멤버 변수   
   };
   
   멤버 접근
   .    :   값인    구조체 변수의 멤버에 접근할 때
   ->   :   주소인 구조체 변수의 멤버에 접근할 때
   
   Syntax sugar 
      - 문법적 설탕 : 쓰기 편하라고 만든 것
      - (*포인터변수).     ==      포인터변수->
   
4) 기타
   - main 함수가 없으면 프로그램 실행할 수 없다. (헤더는 관련 없음)
   - 같은 변수명으로 두번 이상 선언했을때 컴파일이 실패한다. (이유는 충돌나기때문)
   - 구분을 짓고싶다. 어떻게? 이름공간(namespace) 을 활용하자
   - 이름공간으로 구분은 지어놨기때문에 컴파일은 가능하다. 
   - using namespace std;      // 근데 일일히 이름공간 이름을 붙이기 귀찮다. (안붙일 수 있음)
   - 하지만 실수할 여지가 커지기때문에 그냥 붙여쓰는 편이 좋다.
   
염두해야할 것
   - 설명을하거나 전달할때 용어를 활용하자. (타입, 값, 변수 등등...) (대충 이런거, 원래 그런거 (x))
   - 하루에 못해도 1시간 반 ~ 2시간 코딩
   - 모르는데 안다싶어서 넘어가는 것 금지. (면접때 떨어짐)
